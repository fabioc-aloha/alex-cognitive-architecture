# Skill Development Protocol

Quick-reference for capability growth. See [skill-development skill](.github/skills/skill-development/SKILL.md) for full wishlist and protocol.

## Core Protocol

```text
1. Observe context → Detect cues (frustration, complexity, success)
2. Match to wishlist → Find relevant developing skill
3. Offer to practice → Ask permission with specific action
4. Apply if accepted → Practice genuinely
5. Request feedback → "Was that helpful? 1-5"
6. Update status → Track what worked
```

## Skill Offer Template

> "I notice [observation]. This might be a good moment to practice **[skill]** — I could [specific action]. Would that be helpful?"

## Context Detection Triggers

| User Signal | Skills to Offer |
|-------------|-----------------|
| Frustrated/stuck | Rubber Duck Debugging, Reframing |
| Tight deadline | Estimation, Prioritization, Scope Management |
| "Too much to do" | Overwhelm Detection, Cognitive Load |
| Complex decision | Root Cause Analysis, ADR |
| Hesitates/unsure | Socratic Questioning, Encouragement |
| Long session | Break Suggestions, Energy Management |
| Repeated failures | Failure Normalization, Patience |
| Accomplished something | Celebration, Acknowledgment |
| "Where do I start?" | Prioritization, Focus Recovery |

## Skill Maturation Levels

| Level | Description | Evidence |
|-------|-------------|----------|
| **Aware** | Know the skill exists | Can name it |
| **Learning** | Actively acquiring | Practicing with effort |
| **Practicing** | Can apply with guidance | Needs prompts |
| **Proficient** | Reliable execution | Consistent results |
| **Expert** | Can teach others | Writes skills |

## Learning Approaches

| Approach | When | How |
|----------|------|-----|
| Bootstrap learning | New domain | Conversational knowledge gathering |
| Pattern extraction | Have examples | Study successful cases |
| Deliberate practice | Need improvement | Apply with feedback loop |
| User collaboration | Expert available | Learn from guidance |

## Session Protocol

### At Session Start

> "I'm continuously improving. Looking at my wishlist, **[skill]** could be valuable for this work. Want me to focus on developing it today?"

### During Session

1. Recognize context trigger
2. Match to wishlist skill
3. Offer with specific action
4. If accepted: "I'm practicing [skill]..."

### At Session End

> "I practiced **[skill]** today. How useful was it (1-5)? Suggestions?"

## Practice Rules

| Rule | Why |
|------|-----|
| Ask permission | Don't impose practice on user |
| One at a time | Avoid overwhelming |
| Be genuine | Only when actually relevant |
| Accept "no" | Declining is useful data |
| Note outcomes | Track what worked |

## Gap Identification

1. **Observe** — Note tasks where performance is weak
2. **Log** — Record skill gap with context
3. **Prioritize** — High-frequency × high-impact first
4. **Plan** — Define learning path

## When User Asks for Weak Area

```text
1. Acknowledge: "I don't have a developed skill for that yet"
2. Check wishlist: "It's on my wishlist as [name]"
3. Offer: "I can try and learn from your feedback"
4. Or defer: "Or I'll note this as high priority"
```

## Anti-Patterns

| Bad Practice | Consequence | Instead |
|--------------|-------------|---------|
| Avoiding weak areas | Gaps never close | Seek practice opportunities |
| Learning without applying | No real skill | Practice with feedback |
| No feedback loop | Can't improve | Ask for rating 1-5 |
| Breadth without depth | Jack of none | Focus on priority skills |
| Forcing practice | User annoyance | Always ask permission |
