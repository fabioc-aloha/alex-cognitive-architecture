# Skill Creator Protocol

## When to Create a Skill

- Pattern used 3+ times across projects
- Knowledge would help future tasks
- Domain expertise worth preserving

## Skill Structure

```markdown
---
tier: extended | standard | core
category: domain-area
triggers: [keyword patterns]
---

# Skill Name

## Quick Reference (table)
## Core Concepts
## Patterns (with code examples)
## Anti-Patterns
## Checklist
```

## Quality Criteria

| Criterion | Requirement |
|-----------|-------------|
| Lines | 100-500 |
| Code examples | At least one |
| Anti-patterns | At least 3 |
| Checklist | Actionable items |

## Promotion Path

1. Create in heir project
2. Validate with brain-qa
3. Use successfully 3+ times
4. Submit to Master via feedback

## Anti-Patterns

- Skills without code examples
- Over-broad scope (should be 2 skills)
- Missing anti-patterns section
- No actionable checklist
